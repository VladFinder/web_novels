# Резюме реализованных функций

- **Выбор эмоции**: если эмоция уже сохранена на сегодня, показываем выбранную; иначе — экран выбора. Сохранение жестко ограничено одной эмоцией в день.
- **Привязка к Telegram ID**: ID берется из `Telegram.WebApp`, для дев-режима есть безопасный fallback (`getSafeTelegramId`, локальный ID в `localStorage`).
- **Основной экран**: приветствие, случайная поддерживающая фраза под выбранную эмоцию, кнопки календаря и тестовых историй (истории видны только whitelisted ID).
- **Календарь**: подсветка дней с эмоциями, модалка с детальной эмоцией/заметкой/мыслями, навигация по месяцам, поле «мысль за сегодня» с сохранением через API.
- **API-клиент**: `src/services/apiClient.js` — единая точка работы с `/api/users`, `/api/emotions`, `/api/thoughts`. Даты нормализованы в `YYYY-MM-DD`.
- **Темы и иконки**: `src/constants/emotions.js` хранит названия, иконки, мини-иконки, градиенты и фразы для каждой эмоции.

## Кратко по данным
- Эмоции хранятся в `server/data/<telegramId>/<yyyy-mm-dd>.json` (см. `server/json-server.js`).
- Мысли за день пишутся в `server/data/emotions.json` в блок `user_thoughts` (совместимость с текущим сервером).

## Быстрый флоу
1. `EmotionSelect` → `getEmotionByDate()` на сегодня → выбор эмоции → `saveEmotion()`.
2. `MainScreen` → берет эмоцию из стора/сервера → показывает фразу, открывает истории/календарь.
3. `EmotionCalendar` → `getEmotionsRange()` для всех дат → отображение сетки, модалка, `saveThought()` для заметки за сегодня.
